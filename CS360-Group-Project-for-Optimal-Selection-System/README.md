# CS360-Group-Project-for-Optimal-Selection-System

Optimal Samples Selection System v2
=====================================

1. 增量覆盖追踪 - O(1) 评估扰动可行性
2. 更强的特征工程 - 12维特征，更高区分度
3. 改进的局部搜索 - 冗余检测 + remove-repair 策略
4. 有效的模拟退火 - 可行性感知邻域 + 自适应温度
5. ILP 精确求解（小规模）- 验证和基准对比
6. 多次重启取最优



整体流程分为几个阶段。冷启动阶段（前100-200步）用随机策略尝试各种扰动，积累训练样本。学习阶段开始训练神经网络，但仍然以一定概率（比如20%）做随机探索来保证样本多样性。加速阶段（样本充足后）主要依赖网络预测来选择扰动，搜索效率大幅提升。
这个方案的优势在于：不需要预先准备训练数据、不依赖外部大模型、网络很小（几百个参数）推理极快、对搜索质量是纯加速而非替代。同时它保持了正确性保证——无论网络预测得好不好，最终输出的解都会经过严格的覆盖验证，网络只影响搜索速度，不影响结果正确性。

BitSetUtils（位运算工具） 用整数二进制位表示集合，把交集运算变成位与操作，速度比用Python的set快几个数量级。
CoverageInstance（问题建模） 接收五个参数，生成所有目标子集和候选分组，预计算覆盖关系矩阵——哪个分组能覆盖哪些目标。
GreedySolver（贪心） 每步选覆盖最多未覆盖目标的分组，快速得到一个基础解。
MiniNeuralNet（手写神经网络） 8→16→8→1的三层网络，纯Python实现，零外部依赖。包含前向传播、反向传播、在线学习——边搜索边训练。
FeatureExtractor（特征提取） 从当前解和候选扰动中提取8维特征：独占覆盖率、重叠度、冗余度、搜索进度等。
NeuralGuidedLocalSearch（核心创新） 三阶段策略——冷启动阶段随机收集数据，学习期70%用网络引导，成熟期90%用网络引导。网络预测每种扰动的改进概率，优先尝试最有希望的操作。
SimulatedAnnealing（模拟退火） 在局部搜索结果上进一步优化，允许概率性接受更差的解来跳出局部最优。
ResultDatabase（数据库） 按要求的命名格式保存和管理运行结果
